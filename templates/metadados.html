1. Fonte e Tamanho do Dataset

A fonte dos dados utilizados um conjunto de dados extraído diretamente do LinkedIn, disponibilizado no Kaggle. O dataset contém informações sobre ofertas de emprego publicadas no LinkedIn durante o mês de abril de 2024 nos Estados Unidos.

O dataset original é um ficheiro formato .csv que possui 123.849 registros e 31 colunas.

1. Frameworks utilizados

Foram utilizados diversos frameworks e bibliotecas para realizar esta análise de dados, cada um com funções específicas e complementares:


Pandas foi usado para manipular e analisar os dados tabulares. Com a sua estrutura principal, o DataFrame, foi possível organizar, filtrar e transformar os dados de forma eficiente, simplificando todo o processo de análise.

NumPy foi essencial para operações matemáticas e manipulação de arrays, garantindo a rapidez e precisão nos cálculos, especialmente em grandes conjuntos de dados numéricos.

Matplotlib foi utilizado para criar gráficos personalizados e detalhados. Esta biblioteca permitiu construir visualizações de dados que facilitam a interpretação dos resultados, desde gráficos simples até representações mais complexas.

Seaborn foi utilizado para criar gráficos estatísticos e visualizações de dados. Com a sua biblioteca de visualização de dados, foi possível criar gráficos estatísticos e visualizações de dados que facilitam a interpretação dos resultados, desde gráficos simples como de pizaa até representações mais complexas como o mapa interativo.

OS foi utilizado para interagir com o sistema operativo, permitindo manipular diretórios e ficheiros de forma eficiente, o que garantiu uma organização consistente durante o desenvolvimento do projeto.

3. Limpeza dos Dados

Antes de iniciar a análise exploratória, foi necessário realizar o processo de limpeza dos dados para garantir que eles estivessem prontos para análise. O processo de limpeza incluiu as seguintes etapas:


Remoção de Colunas Irrelevantes: Diversas colunas foram removidas do dataset, pois não eram necessárias para a análise, como job_id, company_name, company_id, original_listed_time, pay_period, job_posting_url, entre outras. Essas colunas não agregavam valor à análise proposta.


Tratamento de Valores Ausentes: Algumas colunas do dataset continham valores ausentes. Para lidar com isso:

A coluna views foi preenchida com o valor 0.

• A coluna applies foi preenchida com o valor 0.

• As colunas formatted_experience_level e skills_desc foram preenchidas com “Não especificado” e “Habilidades não especificadas”, respectivamente.


Remoção de Duplicatas: Foi realizada a remoção de duplicatas com base nas colunas title, description e location. As vagas duplicadas foram eliminadas para garantir que cada vaga fosse única.


4. Processamento dos Dados

Com os dados já limpos, as etapas de processamento envolvem a análise dos dados de forma a extrair informações relevantes para a análise. O processo de processamento incluiu:


4.1 Contagem de Vagas por Mês

A primeira análise realizada foi para verificar a distribuição das vagas ao longo do mês de abril de 2024. A coluna listed_time foi utilizada para extrair a data de publicação de cada vaga e a quantidade de vagas por mês foi calculada. O resultado mostrou que todas as vagas eram de abril de 2024.


4.2 Análise de Trabalho Remoto

A coluna remote_allowed foi utilizada para analisar a permissão de trabalho remoto. As vagas foram agrupadas conforme as categorias “Permitido” e “Não Permitido” para entender a proporção de vagas que permitem trabalho remoto..


4.3 Relação entre Vistas e Aplicações

Foi calculada a correlação de Pearson entre as colunas views e applies para verificar a relação entre o número de visualizações e o número de candidaturas. A análise foi realizada separando as vagas em duas categorias: “Remoto Permitido” e “Não Remoto Permitido”.


4.4 Contagem de Vagas por Tipo de Trabalho

A análise das vagas por tipo de trabalho foi realizada utilizando a coluna formatted_work_type. A distribuição das vagas foi calculada entre os tipos full-time, contract, part-time, entre outros.


4.4 Contagem de Vagas por Localização

As vagas foram segmentadas por localização com base na coluna location. Foi calculado o número total de vagas por estado, permitindo identificar as regiões com mais ofertas de emprego.


4.5 Análise de Nível de Experiência

Foi realizada uma análise do nível de experiência exigido para as vagas, utilizando a coluna formatted_experience_level. As vagas foram agrupadas conforme o nível de experiência exigido.


4.6 Análise de Títulos de Vagas mais Ofertados

Foi calculado o número de candidaturas para cada título de vaga listada entre os 10 títulos mais ofertados permitindo observar os títulos com maior presença na plataforma.

Essa comparação possibilita entender se as vagas mais anunciadas também atraem um maior volume de candidatos.


4.7 Análise das Vagas mais Aplicações

Para esta análise foi realizada a soma total de candidaturas foi calculada para os 10 cargos com mais interesse por parte dos candidatos e calculado o número de ofertas para cada título de vaga presente na lista das mais candidatas. Por fim, foi avaliado o percentual de aplicações em relação ao número total de ofertas para verificar a atratividade dessas vagas no mercado.


4.8 Habilidades Mais Frequentes

As habilidades exigidas nas ofertas de emprego foram extraídas da coluna skills_desc. As habilidades foram divididas em listas, e as 15 habilidades mais frequentes foram identificadas e listadas para observar as competências mais exigidas nas ofertas de emprego.


5. Visualizações

Diversas visualizações foram criadas para ilustrar os resultados das análises realizadas. Dentre as visualizações estao:


• Gráfico de barras para a distribuição das vagas por tipo de trabalho.

• Mapa interativo com a quantidade de vagas por localização

• Gráfico de pizza para permissão de trabalho remoto.

• Gráfico de dispersão mostrando a relação entre vistas e aplicações.

• Nuvem de palavras com as 15 habilidades mais frequentes..
